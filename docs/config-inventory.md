# Config inventory — where every machine-level config actually lives

**Purpose:** a durable map of all the AI-tool, shell, SSH, and cloud config
spread across this workflow, so no future session (human or AI) has to
rediscover it. Written 2026-07-10 after a session found the same facts the hard
way.

**One-line summary:** config is currently spread across **three overlapping
sync systems plus several things synced by nothing**. This repo (`ai-devops`) is
the intended long-term hub; see [`config-consolidation-proposal.md`](config-consolidation-proposal.md)
for the plan to converge everything here.

> ⚠️ **No secrets in this doc.** Secret-bearing files are named by type only.
> Real secrets live in 1Password (`vibe_coding` vault) and in machine-local files
> that are never committed.

## The map

| Location | What it holds | Synced across machines by |
|---|---|---|
| **`ai-devops` repo** (git; ~1.5 GB incl. chat archives) | Skills → `~/.claude/skills`; global instructions → `~/.claude/CLAUDE.md`; Codex global instructions; workflow model config templates; Claude/Codex chat archives | ✅ git + `install.sh` / `bin/ai-install-skills` (Windows + Ubuntu) |
| **Dropbox `\vibe coding\ssh keys\`** | `master_setupsshwindows.ps1` → writes `~/.ssh/config` (all host aliases) + the `916-alien` private key + optional cloudflared | ⚠️ manual script; **embeds a plaintext private key** |
| **Dropbox `\vibe coding\…MCP servers\`** | `setup-claude-mcps.ps1` / `setup-codex-mcps.ps1` → write MCP server entries into Claude/Codex config | ⚠️ manual scripts; **embed secret tokens** |
| **`~/.claude/settings.json`** | MCP server list **+ live secret tokens** + push-notif / auto-update prefs | ❌ generated by the Dropbox MCP script (not itself synced) |
| **`~/.codex/config.toml`** | Codex prefs (`model`, `model_reasoning_effort`, `[windows] sandbox`, desktop UI, enabled plugins, local `node_repl` MCP) **+ machine-specific runtime paths** | ❌ nothing |
| **`~/.claude/projects/*/memory/`** | Auto-memory (per-project `MEMORY.md` + fact files) | ❌ **nothing — real gap** |
| **gcloud config** (`%APPDATA%\gcloud` / `~/.config/gcloud`) | Default project/region for `gcloud` | ❌ per-machine only |
| **1Password `vibe_coding` vault** | The actual secrets (tokens, keys, DB creds, logins) | ✅ correctly centralized |
| **`/etc/ai-devops/*.env`** (Ubuntu) | Real workflow model commands + paths | ❌ machine-local by design (never committed) |

## Detail per location

### ai-devops repo (the intended hub)
- `skills/claude/*` is the **source of truth** for `~/.claude/skills` on every
  machine; `bin/ai-install-skills` installs them. `skills/codex/*` mirrors this
  for Codex.
- `templates/system/CLAUDE-global.md` → each machine's global `~/.claude/CLAUDE.md`;
  `templates/system/AGENTS-global-codex.md` → Codex global instructions;
  `templates/system/machine-atlas.md` → the per-machine environment atlas.
- `config/*.env.example` seeds `/etc/ai-devops/` **without clobbering** existing
  machine-local values.
- `claude_chats/` + `codex_chats/` are transcript archives (large, may contain
  secrets — never load into AI context).

### Dropbox scripts (two separate systems)
- **SSH:** `master_setupsshwindows.ps1` writes `~/.ssh/config` with every host
  alias (coolify/vps2/seafile/edge1/edge2/backupwiz/comp/auth/vpn/…), drops the
  `916-alien` key, fixes Windows ACLs, optionally installs cloudflared. **SSH
  only** — it touches nothing else (not Claude, Codex, skills, gcloud, memory).
- **MCP:** `setup-claude-mcps.ps1` / `setup-codex-mcps.ps1` generate the MCP
  server blocks in Claude/Codex config, including auth tokens.

### settings.json / config.toml (mostly not portable as-is)
- `~/.claude/settings.json`: MCP servers + **live tokens** + a few prefs. Only
  the non-secret shape is portable; the tokens must come from 1Password.
- `~/.codex/config.toml`: only ~5 preference lines are portable
  (`model`, `model_reasoning_effort`, `[windows] sandbox`, a couple desktop
  prefs). The rest is per-install runtime paths — **do not sync wholesale.**

### Gaps synced by nothing
- **Memory** (`~/.claude/projects/*/memory/`) — highest-value gap: a memory
  written on one machine never reaches the others.
- **gcloud defaults** — must be set per machine (dflow: project
  `lithe-breaker-323913`, region `us-east4`).
- **Portable Codex prefs** — the handful of lines above.

## Security landmines (why naive git-sync is unsafe)
1. **`master_setupsshwindows.ps1` stores the `916-alien` private key in plaintext**
   in Dropbox.
2. **`~/.claude/settings.json` stores live tokens in plaintext** (a 1Password
   service-account token, a Trigger PAT, MCP bearer tokens).

Neither may enter a git-synced dotfiles system as-is. The consolidation plan
pulls all secrets from 1Password at install time so nothing secret is committed.
