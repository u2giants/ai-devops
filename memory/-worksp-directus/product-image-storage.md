Product `cover_url` images originate from ClickUp attachments (workspace 2298436, CDN `t2298436.p.clickup-attachments.com`). The first import (`pm-system/migration/clickup-images.mjs`) stored ClickUp's `thumbnail_large` URL (the `<uuid>_large.jpg` form). ClickUp later **disabled that thumbnail endpoint**, so those URLs now return **HTTP 403**. The full attachment `url` (the `<uuid>/<filename>` form) is still publicly served and works — it has no signature/expiry, so it is a stable public CDN link.

**Migration complete (2026-06-12):** 16,534 products, all with `external_id`. Final: **3,747 covers on Spaces** (originals + thumbs), **0 ClickUp URLs remain**, 12,787 with no image (2 unrecoverable ClickUp 403s → set empty). Transient Directus 502s / ClickUp nulls during the run left a few stragglers; mopped up with `SKIP_EMPTY=1` re-runs (skips the 12.8k empties).

**Migration (single script):** `pm-system/migration/clickup-to-spaces.mjs`. For each product it downloads the **original** ClickUp cover (no resize) and uploads it verbatim to the public Space, repointing `cover_url` → `https://poppim.nyc3.digitaloceanspaces.com/covers/<id>.<ext>`. Source resolution: if `cover_url` is already a working ClickUp URL, download it directly; otherwise re-derive via the ClickUp API (`external_id`) — covers the dead `_large` thumbnails and the empties (re-checked). Resumable via `/tmp/clickup-to-spaces.checkpoint`; ClickUp calls rate-limited ~85/min. Run: `POPPIM_ENV_FILE=/home/ai/.directus-deploy.env DX_URL=https://[retired-hostname-removed] node pm-system/migration/clickup-to-spaces.mjs`. **User directive 2026-06-12: do NOT resize — store originals regardless of size** (an earlier sharp→webp≤1000px version was reverted; the 1,589 webp it wrote are being replaced with originals).

**Frontend:** Spaces stores BOTH the verbatim original (`covers/<id>.<ext>`) and a small webp thumbnail (`covers/<id>_thumb.webp`, ≤400px) generated by the migration. poppim-web `adapter.ts` derives `coverThumbUrl` from `cover_url` (when it's a Spaces URL, swap to `<id>_thumb.webp`; else fall back to the full URL). The kanban card (`PimTaskCard.tsx`) loads `coverThumbUrl` (falls back to the full original on a 404), and the opened `TaskDetailModal.tsx` loads the full original (`coverUrl`, object-contain). All `<img src>` are unauthenticated, so the host MUST serve from a **public** URL — Spaces objects are uploaded `public-read`. See [[feedback-store-originals]].

**Durable target decision (user, 2026-06-12): DigitalOcean Spaces (S3-compatible).** NOT Cloudflare R2 (only Seafile buckets exist there, off-limits) and NOT Directus local files. The durability pass will read each recovered `cover_url`, download the bytes, upload to a public DO Space, and rewrite `cover_url` to the Space URL — no second ClickUp crawl needed. **No DO Spaces credentials exist on this machine yet** (checked `.directus-deploy.env`, `hiclaw-manager.env`); they must be provided (access key, secret, region/endpoint, space name).
