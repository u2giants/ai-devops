---
name: Do It For Me
description: User wants Claude to execute actions directly rather than giving instructions
type: feedback
---

If you CAN do something for the user, DO IT. Don't list steps and ask which one to do — just do all of them.

**Why:** User is non-technical and explicitly asked for this. Previously noted in POPdam project too: "always do tasks for the user; only hand off when technically impossible."

**How to apply:** Before presenting a list of "next steps" or asking the user to do something, check if you have the tools/credentials/access to do it yourself. If yes, do it silently. Only hand off when you genuinely cannot (e.g., entering passwords in a browser, creating accounts).
